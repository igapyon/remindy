# MinuteTicker.java

## Overview
`Timer` を用いて 1 分周期のタスク実行環境を提供するクラス。登録済み `MinuteCommand` を順に呼び出し、起動直後は `StartupCommand` を即時実行します。実際には予定時刻の約10秒前をターゲットに実行されるため、通知なども少し早めに発火します。

## Key Behavior
- 現在時刻から次の 00 秒までの遅延を計算し、以降 60,000ms 間隔で `TimerTask` を起動。
- コマンド登録順に実行するため、順序依存の処理も制御可能。
- クラス名が `StartupCommand` のものだけは初回遅延前に別ループで呼び出し、通常巡回からは除外。
- 例外は個別に捕捉し、他コマンドへの影響を最小化。

## Notes
- `Timer` はデーモンスレッドで動かすため、メインスレッド維持は呼び出し側に任せています。
- リード時間 (`LEAD_SECONDS`) は 10 秒に固定されており、分刻みの予定よりわずかに早くコマンドへ時刻を渡します。
